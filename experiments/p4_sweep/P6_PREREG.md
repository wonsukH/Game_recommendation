# P6 OOD 재실험 — 사전등록 프로토콜 (v1, 2026-07-06 자율운행 중 작성)

> **유형**: design-spec · **상태**: active · **갱신**: 2026-07-14 — 사전등록 프로토콜(OOD 관측 **전** 고정, 등록 후 append-only). **수정안 v3까지 사용자 승인 완료(2026-07-14) — 실행 개시.** 이후 변경 금지(관측 전 마지막 수정).

## 목적
P4 shortlisting은 편향 코호트(리뷰어-스노볼)에서의 잠정 순위다. P6는 **de-biased 패널에서 리더 구성을 재실험**해 최종 확정한다 — winner's curse의 confirmation 단계. fresh-panel(T29, n=586)이 준-OOD 신호를 이미 줬지만 스노볼-연결이라 완전 독립이 아니다.

## 사전등록 가설 (감사 T32~T33로 재작성 — v2)
- **H1(주가설, 랭커 — T36 정정)**: **EASE(λ≈100)** 가 OOD에서 프로덕션 condcos와 userknn을 **주지표 A에서 유의 격파**한다(in-cohort EASE−userknn +0.061 SIG; condcos 양축 최하). → 성립 시 P5 서빙 랭커=EASE. *구 H1의 "userknn 최고"는 cutoff 버그였음(T35). OOD 랭커 셋에 EASE(λ 50/100/200)·userknn·knnpd03·condcos 포함.*
- **H2(선호)**: pvalue와 pctl_game은 주지표 A에서 **구별 불가**하다(귀무 유지 예상). 어느 하나가 target-독립 지표 B에서 유의하면 그것을 선호로 채택, 아니면 **최단순 pctl_game**.
- **H3(knnpd03 knob)**: knnpd03이 순정 userknn 대비 **target-독립 지표 B(wishlist)에서 유의**를 OOD에서 유지하면 발굴 knob 채택(현 in-cohort **+0.0073 SIG**, 웨이브2), 붕괴하면 순정 userknn. *지표 A/SNIPS/judge는 순환이라 이 판정에서 제외.*
- **반증 조건**: H1 붕괴(랭커 교체 근거 상실) → condcos 유지. H3의 지표 B ns 지속 → knnpd03 폐기, 순정 userknn 확정.

## 패널 설계
- **소스**: 랜덤 accountID 프로브(SteamID64 = 76561197960265728 + U[0, ~1.6e9] 균등 샘플) → 공개 프로필 + 라이브러리 12+게임 필터. 스노볼 링크 무관 = 코호트 독립.
- **크기**: 최소 300명(CI 폭 fresh-panel 수준), 목표 500.
- **그래프**: 기존 train 1,133 고정(순수 일반화) **및** OOD-혼합 그래프(절반 편입) 2셋 — 후자는 "데이터가 저럴 때 배포하면"의 근사.
- **1회성**: 이 패널은 확정 판정에 1회만 사용. 반복 조회 금지.

## 지표 (감사 T32~T33로 보강 — v2)
> **감사 교정**: v1은 graded NDCG(=build_relevance)를 단일 주지표로 뒀으나, 그 타깃은 선호와 playtime 백분위를 공유(per-game rho 0.957)해 **순환**. in-cohort NDCG/SNIPS/masked/judge가 전부 이 타깃을 재사용하므로 순환을 반박 못함. → **target-독립 co-주지표를 추가하고, knnpd03·선호 우열은 이것으로만 판정**한다.
- **co-주지표 A (순환 축, 유지)**: graded NDCG@20 — playtime-백분위 정답. paired bootstrap CI.
- **co-주지표 B (target-독립, 신규·knnpd03 판정 기준)**: **held-out wishlist recall@20** (컷오프 T 이후 추가된 미보유 wishlist를 카탈로그 랭킹에서 발굴 — playtime provenance 없음). 시간분할로 누수 차단(입력=T 이전 owned, 정답=T 이후 wishlist). paired bootstrap S0-vs-S1, S0-vs-S2.
  - *현 in-cohort 값*: dev T33(n=178) +0.0056 ns → **웨이브2 T34(n=179, 풀 39.7k) +0.0073 [+0.0011,+0.0142] SIG**. in-cohort 비순환 근거는 확보 → **OOD에서 유지되면 knnpd03 채택**(붕괴하면 순정 userknn).
- **보조**: SNIPS recall(클립 10)·recall@20·null 분리. **단 SNIPS·judge-discovery는 pop-debias 축이라 knnpd03에 by-construction 유리 → knnpd03 판정의 독립 근거로 쓰지 않음**(확인용만).
- **다중비교**: 헤드라인 페어에 BH-FDR 적용(T33에서 하니스 audit_fdr.py 확립).

## judge 교차검증 (질감 축)
- Sonnet blinded 쌍대(기존 프로토콜: grounded 카드, A/B 랜덤화, fit/discovery) n≥16 + **Gemini(flash급) 동일 페이로드 독립 실행** → 모델 간 일치도(cohen's κ) 보고. 페이로드 생성기는 `judge_payload.py`(JUDGE_VARIANT 확장)로 재사용.
- judge는 보조 축: 주축 판정을 뒤집지 않고, S0-계열 내 동률(β 0.2 vs 0.3 등) 타이브레이커로만.

## 실행 비용 추정
랜덤 프로브 히트율(공개+12게임) ~5-10% 가정 시 500명 확보 ≈ 5k~10k 프로브 콜 + 심층 ~170콜/명 ≈ 90~100k콜(약 하루 예산). allowlist 축소 채택 시 대폭 감소.

## 수정안 v3 (2026-07-14 — OOD 관측 전, 사용자 승인 완료; append-only)

> 배경: depth=-1 무편향 풀이 이미 충분(공개+완료 9,742명, 유효항목 ≥12 적격 ≈3,416명) → 프로브 크롤 불필요.
> 2026-07-14 사용자가 실행 계획 전체(A1~A8 + 아래 확정 결정)를 승인. **이 섹션이 관측 전 마지막 수정이다.**

### A1 — EASE 슬롯 추가 (H1 반증 가능성 복원)
v2 H1은 EASE 격파를 주장하나 고정 슬롯 표(S0a~S4)에 EASE가 없어 등록된 평가 셋 안에서 H1이 반증 불가였다. 추가:
| 슬롯 | 구성 |
|---|---|
| S5a / S5b / S5c | pvalue_eb × **EASE(λ=50 / 100 / 200)** — 공정 리스트 빌더 `ease_reclist`(`ease_recheck.py:38`, cutoff 버그 없음) |
**H1 판정은 S5b(λ=100)만 사용.** λ50/200은 민감도 전용 — 사후 최량 선택 금지.

### A2 — OOD relevance 축퇴 명문화
depth=-1 코호트는 업적 행 0건(07-07부터 수집 OFF) → `build_relevance`의 정답이 **playtime-pctl 단독**으로 축퇴.
차단 사유 아님: 완료율의 유일 실증 가치(+0.0073)는 target-독립 지표 B가 별도로 측정하고, 지표 A는 원래 playtime과 rho 0.957 순환. per-game 백분위는 p6 스냅샷 전체(혼합 코호트)에서 계산(기존 코드 의미론 유지, 참조 표본 최대).

### A3 — 패널 재정의 (프로브 절·비용 절 무효화)
- 패널은 **이미 크롤된** depth=-1 풀에서 추출 — 프로브 비용 추정(90~100k콜) 절은 무효.
- 적격 = **p6 스냅샷 기준 유효 플레이 항목 ≥12** (`get_panels`의 min_items 의미론; "raw 보유 ≥12"가 아님).
- 추출: 전체 적격 ≈3,416에서 시드 고정 균등 무작위. **위시리스트 보유를 조건으로 걸지 않는다**(걸면 몰입 유저 쪽 편향). 지표 B는 패널 내 B-적격 부분집합(날짜有 미보유 위시 ≥3, ≈58%)에서 계산.
- **N = 1,000 (사용자 확정; B-부분집합 기대 ≈580)** + **예비(reserve) 패널 500명 별도 격리** — 예비는 향후 *신규 사전등록* 하에서만 사용(예: E5 융합 승자 확정).

### A4 — OOD-혼합 그래프 재정의
v1 "패널 절반 편입"을 폐기하고 **혼합 그래프 = train-1,133 ∪ 탐색 풀 전원**(패널·예비와 결정적 분리)로 대체.
근거: 패널이 두 그래프 조건 모두에서 완전 홀드아웃 → 동일 유저 paired 비교, 검정력 반감 없음. **주 판정 = 동결(train-1,133) 그래프**; 혼합은 배포 근사 보조(판정 못 뒤집음).

### A5 — 지표 B 조작적 정의 고정
`owned`에 취득 시각이 없어 문자적 시간분할(입력=T이전 owned)은 재구성 불가. 지표 B는 검증된 기존 구현(`audit_verify.py`) 그대로: **정답 = 최근 ≤10개 풀 내 미보유·날짜有 위시 추가분(≥3 필수), 입력 = 전체 플레이 프로필, 위시는 입력에 절대 불사용**(구조적 무누수). T34 기준치(+0.0073)와 비교 가능성 유지. OOD 위시 행은 패널 동결 시점에 스냅샷(`outputs/p6/wishlist_ood.pkl`) — 라이브 DB 성장과 무관.

### A6 — judge 범위
Sonnet blinded 쌍대 n≥16만 자율 실행. **Gemini κ 교차는 사용자 배석 세션으로 연기**(유료 자율 사용 금지 제약, decisions.md). 보조축 — 주 판정 못 뒤집음(기존과 동일).

### A7 — FDR 패밀리 고정 (m=8, 동결 그래프)
헤드라인 페어를 다음 8개로 고정, BH-FDR 적용. 그 외 모든 비교는 서술적(판정 불사용):
① S5b vs S1 [NDCG] (H1) ② S5b vs S4 [NDCG] (H1) ③ S1 vs S2 [NDCG] (H2) ④ S1 vs S2 [wishlist] (H2)
⑤ S0b vs S1 [wishlist] (H3) ⑥ S0a vs S1 [wishlist] (H3 그리드) ⑦ S0b vs S1 [NDCG] (해악 점검) ⑧ S5b vs S1 [wishlist].

### A8 — 데이터 동결
추출 1회(`outputs/p6/`)가 드라이런·확정·탐색 전 작업의 불변 기질. 크롤은 계속 돌아도 P6에 무영향. 동결 전 1회성 게임 메타 보충 크롤(미분류 1,012 appid, ≈1k콜)을 실행(사용자 확정) — 풀 4.5% 억제 방지.

### 확정 실행 절차 결정(사용자 확정 2026-07-14)
- 확정 패널은 **1회성**(기존 조항 유지) — 하네스 검증 게이트 V1~V6(구코호트 재현·앵커 분리·T34 재현·방화벽 단언·결정론·메모리) 전부 통과 + **탐색 풀 드라이런 리더보드 사용자 사인오프 후에만** 실행.
- 탐색 트랙 E1(코호트-시프트)·E2(무편향 인기도/SNIPS 재교정)·E3(라이트 유저 5–11 기술통계)·E4(포화곡선) 승인 — 전부 탐색 풀 전용, 확정·예비 패널 접촉 금지(`assert_firewall`). E5(EASE 정밀 그리드+융합 2종)는 H1 성립 시만, 서빙 반영은 예비 패널+신규 사전등록 경유.
